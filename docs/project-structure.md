# Project Structure

This template follows a `src/`-based layout and includes common tooling for modern Python projects.

A typical layout looks like:

```text
├── .venv/                     # Virtual environment (excluded from git)
├── src/                       # Package source code
│   └── your_actual_project_name/
│       ├── __init__.py
│       └── ...
├── tests/                     # Test suite
├── docs/                      # Additional documentation
├── .gitignore
├── .aider.conf.yml            # Aider configuration (if used)
├── requirements.in            # Direct dependencies
├── requirements.txt           # Locked dependencies (generated)
├── tasks.py                   # Invoke tasks
├── README.md                  # Project overview and quick start
└── pyproject.toml             # Build backend and tool configuration
```

## Key components

### `src/`

Contains the actual Python package for your project.

- `src/your_actual_project_name/__init__.py` – Top-level package module.
- Additional modules and subpackages live under this directory.

Using a `src/` layout helps avoid certain import pitfalls and ensures tests import the installed package rather than the local source by accident.

### `tests/`

Contains the test suite, typically mirroring the structure of `src/`.

- Test files usually follow the pattern `test_*.py` or `*_test.py`.
- Pytest discovery is configured in `pyproject.toml` under `[tool.pytest.ini_options]`.

Example:

```python
from your_actual_project_name import hello


def test_hello_returns_expected_greeting() -> None:
    assert hello("world") == "Hello, world!"
```

### `docs/`

Holds additional documentation split into focused topics, such as:

- `getting-started.md`
- `customizing-template.md`
- `development-tasks.md`
- `dependencies.md`
- `project-structure.md`
- `tips-and-tricks.md`

This keeps `README.md` concise while still providing detailed guidance.

### `requirements.in` and `requirements.txt`

- `requirements.in` – The list of direct dependencies you care about.
- `requirements.txt` – The fully pinned dependency set generated by `pip-compile`.

These are managed via `pip-tools` and `invoke` tasks.

### `tasks.py`

Defines [Invoke](https://www.pyinvoke.org/) tasks for common operations:

- Installing and syncing dependencies.
- Formatting, linting, and type-checking.
- Running tests.
- Cleaning build artifacts and caches.

See [development-tasks.md](development-tasks.md) for details.

### `pyproject.toml`

Central configuration file for:

- Build system (`[build-system]`).
- Project metadata (`[project]`).
- Tooling configuration:
  - Black (`[tool.black]`)
  - Ruff (`[tool.ruff]`)
  - isort (`[tool.isort]`)
  - mypy (`[tool.mypy]`)
  - pytest (`[tool.pytest.ini_options]`)

### `.venv/`

The recommended location for the project’s virtual environment. This directory is typically excluded from version control via `.gitignore`.

### `.gitignore`

Specifies files and directories that should not be committed to version control, such as:

- `.venv/`
- `__pycache__/`
- Build artifacts
- IDE-specific files

### `.aider.conf.yml` (optional)

Configuration for the [aider](https://github.com/Aider-AI/aider) tool, if you use it for AI-assisted development.

## Adapting the structure

When you customize this template:

- Rename `src/project_name/` to `src/your_actual_project_name/`.
- Update imports and tests to use your new package name.
- Adjust `pyproject.toml` and any other configuration files accordingly.

For a step-by-step guide, see
[customizing-template.md](customizing-template.md).
